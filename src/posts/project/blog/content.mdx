## 적용하기

빌드 & 배포하는 방법은 어렵지 않다.

1. 가이드대로 패키지 설치 후 차근차근 구현하는 방법
2. 소스파일 fork 또는 clone 후 필요한 부분을 찾아서, 수정하는 방법

당연히, 2번의 방법을 택했고 (굳이 변명하자면, 다른 개발할게 좀 많아서... ㅠㅠ)
일부 소스/리소스만 변경해서 사용하게 되었다.

배포는, vercel을 사용했고 현재는 별도 도메인 없이 기본 제공되는 [주소]()로 배포되도록 했다.

### 배포

vercel 사용방법은 별거 없다.

>[vercel](https://vercel.com) - github 연결 - 배포할 repogitory/branch 선택하면 끝!!

만약, 커스텀 도메인이 있다면 프로젝트 설정에 추가해주면 된다.

### 장점

- 글쓰고, commit & push만 하면 자동 배포
- 커스텀이 자유롭고, 필요에 따라 여러가지 기능을 추가할 수 있음
- 미리보기 이런거 없이, 로컬상에서 글 작성시 결과화면을 바로 볼 수 있음
- 동적 path를 사용하고 있어서, 폴더만 작성하면 자동으로 분류됨
- 가볍고 빠름

### 단점

- 웹상에 글쓰기 기능이 없어서, 소스코드 수정하는 방식으로 글을 작성해야함
- draft 기능이 없어서, 작성중 배포가 불가능함
- 동적 path를 사용하고 있어서, output: export을 사용하지 못함 (정적페이지 배포 불가능)

### 일단 배포

- draft 기능만 추가 (제목에 [draft]가 추가된 경우에는 목록에서 노출되지 않도록 함)
